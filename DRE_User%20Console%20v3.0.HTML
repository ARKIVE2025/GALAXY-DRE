WHAT IS THE DRE CONSOLE?

The DRE Command Console is a personal GTM (Go-To-Market) execution engine — built for product launchers, indie builders, and first-time founders who need a structured system to take their product from zero to market.

It is a local-first tool. Nothing is stored on any server. Your data lives on your device. You bring your own AI API key. You are fully in control.

The DRE Framework: Every plan inside this console follows an 8-step execution arc — Setup → Hook → Entry → Moment → Asset → Action → Capture → Expansion. These aren't suggestions. They are the minimum viable moves to launch any product with intention.
THE 4 SECTIONS OF THE CONSOLE
01
GTM PLANS — Your launch playbooks. Each card represents one week of execution for one tool. Click a card to expand the full 8-step plan, checklist, and AI actions.
02
ECOSYSTEM — All the tools in your project workspace. After importing your workspace JSON, every tool in your project appears here as a card you can click to get an instant AI briefing.
03
SOCIAL MEDIA 101 — A foundational crash course on every major platform, algorithm rules, bio links, content types, and why Step 0 (your digital infrastructure) is the actual launch. Read this first if you are new.
04
SAVED RUNS — Every time you click "Save Run" on a plan, it gets archived here. Use this to track your execution history and reload plans you've already worked through.

STEP 1 — WHAT YOU NEED BEFORE YOU BEGIN
⚠ STEP 0 IS REAL: Before you can launch anything, your digital infrastructure must exist. Read Social Media 101 in the console for the full breakdown. The short version: set up your social accounts, write your bios, and create a bio link. If these don't exist, nothing downstream will work.
A
Your Project Details — Have a clear answer to: What is your tool? Who does it help? What problem does it solve? You'll need this to fill out your workspace template accurately.
B
A Text or Code Editor — To fill out your workspace JSON file, you need a basic text editor. VS Code (free, recommended), Notepad (Windows), or TextEdit (Mac) all work. The key is saving the file with a .json extension.
C
An AI API Key — The console's AI Command panel requires a key from one of 3 supported providers. You get this key from the provider's website (links in Step 3). Keys are free or low-cost to start. Your key is saved in your browser only — never sent anywhere else.
D
This Console Open in a Modern Browser — Chrome, Edge, Firefox, or Safari. The console runs 100% locally. No login. No account. No server.
HOW TO OPEN THE CONSOLE
1
Save the DRE_Console.HTML file to your computer (Downloads, Desktop, a project folder — anywhere).
2
Double-click the file. It will open directly in your web browser as a full application. No installation required.

STEP 2 — BUILDING YOUR WORKSPACE JSON

Your workspace is a single .json file that holds all your project data — your tools and your launch plans. You build it once, then import it. You can update it anytime and re-import.

1
Copy the starter template — Use the button below to copy it, then paste it into your text editor and save it as my_project.json (the filename can be anything, the .json extension is required).
2
Fill in your ECOSYSTEM — The ecosystem array is your list of tools. Each tool has: name, color (hex code), status (LIVE or SOON), purpose (one sentence), desc (2-3 sentences), and tier (Free/Pro/etc.).
3
Fill in your PLANS — The plans array is your GTM playbook. Each plan has a week number, a tool name (must match ecosystem exactly), a title, a tagline, 8 execution steps, a checklist, and expansion tools.
4
Validate your JSON — Before importing, paste your file into jsonlint.com to check for errors. A missing comma or unclosed bracket will prevent it from loading.
5
Import it — In the sidebar, click IMPORT WORKSPACE. Select your .json file. The console will load your ecosystem and plans instantly.
STARTER TEMPLATE — COPY THIS
{ "ecosystem": [ { "id": "TOOL_01", "name": "Your Tool Name", "color": "#00d4ff", "status": "LIVE", "purpose": "Short one-sentence purpose statement.", "desc": "A 2-3 sentence description of what this tool does and who it helps.", "tier": "Free / Pro" } ], "plans": [ { "id": 1, "tool": "Your Tool Name", "week": 1, "title": "Week 1 — Initial Launch", "tagline": "The core strategy for the first week.", "steps": { "setup": "What accounts, bios, and infrastructure must exist before you post.", "hook": "The core message or angle to grab your audience's attention.", "entry": "How your target user first interacts with the product.", "moment": "The 'Aha!' moment you need to demonstrate clearly.", "asset": "The actual piece of content to create this week.", "action": "Where and how to post or distribute that content.", "capture": "How you capture the lead or convert the viewer into a user.", "expansion": "How this week connects to next week's strategy." }, "checklist": [ "Set up all social profiles with complete bios", "Create and test your bio link", "Record and post your first piece of content", "Reply to every comment within 24 hours" ], "expansionTools": ["Another Tool Name"] } ] }
MULTI-TOOL EXAMPLE STRUCTURE

If you have multiple tools, add additional objects to the ecosystem array, each with a unique name. Add one plan per tool per week to the plans array, with id values incrementing (1, 2, 3…). The tool name in plans must exactly match the tool name in ecosystem.

STEP 3 — CONNECTING THE AI ENGINE

The AI Command panel on the right side of the console connects to real AI models via API. You choose your provider, paste your key, and the AI becomes a co-founder that knows your entire ecosystem.

Your API key never leaves your browser. It is saved in localStorage on your device only. The only outbound connection is the direct call from your browser to the AI provider's server when you send a message.

FREQUENTLY ASKED QUESTIONS
Q
My workspace won't load. What's wrong?
The most common cause is a JSON formatting error. Paste your .json file into jsonlint.com to find the exact error. Common mistakes: a trailing comma after the last item in an array, a missing quotation mark, or a curly brace that was never closed.
Q
Is my data safe? Where is it stored?
All data is stored in your browser's localStorage — on your device only. Nothing is sent to any server except the AI messages you send to your chosen model's API. Your API key is stored locally and masked after saving.
Q
I updated my workspace JSON. How do I reload it?
Click IMPORT WORKSPACE in the sidebar again and re-select your updated file. This will overwrite the ecosystem and plans in memory and localStorage with your new data. Your checklist states, notes, saved runs, and chat history are preserved.
Q
How many tools and plans can I have?
There is no hard limit. The console renders whatever is in your ecosystem and plans arrays. For best performance, keep ecosystems under 30 tools and plans under 120 entries.
Q
Can I use this on mobile?
The console is designed for desktop (three-column layout). On mobile, the layout will compress. It is functional but the experience is optimized for a laptop or desktop screen at 1280px or wider.
Q
The AI isn't responding. What do I check?
(1) Is your API key saved? The field should show dots. (2) Do you have credits/quota remaining with your provider? (3) Is the correct model selected? (4) For LM Studio: is the app running and a model loaded? Check the browser console (F12) for the exact error message.
Q
What does "STEP 0" mean in every plan?
Step 0 is your digital infrastructure — social accounts, bios, bio links, and basic profile setup. It is named Step 0 because it must be done before any other step. No amount of great content will work if people click your profile and find a blank page. Read the SOCIAL MEDIA 101 section for the full breakdown.
Q
Can I share this console with my team?
Yes — just share the HTML file. Each person will have their own local storage, so their progress, notes, and API keys are separate. For a shared workspace, you can distribute the same workspace .json file so everyone is working from the same ecosystem and plans.
